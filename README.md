# MakeHomePage
教会のクリスマスイベントの参加ホームページを作成する
# 1. システム仕様書

### 1.1 システム概要
*   **名称**: 郡山家庭教会 クリスマス感謝祭2024 参加登録サイト
*   **目的**: イベント情報の周知および、参加希望者の事前申し込みを受け付ける。
*   **ターゲット**: スマートフォンユーザー（QRコードからのアクセスを想定）。

### 1.2 技術スタック (採用技術)
*   **開発環境**: Windows または WSL2 (Ubuntu)
*   **バージョン管理**: Git / GitHub
*   **フロントエンド**:
    *   HTML5 / CSS3 (レスポンシブデザイン：スマホ対応)
    *   JavaScript (ES6+) : アニメーションと通信担当
*   **バックエンド**:
    *   Java (JDK 17 or 21)
    *   Framework: Spring Boot (Webモジュール)
*   **公開・検証ツール**:
    *   ngrok (ローカルサーバーの外部公開用)

### 1.3 機能要件 (何ができるか)

#### 【フロントエンド（画面）】
1.  **情報表示機能**:
    *   ポスターの内容（日時、場所、出し物、ブース案内）を表示。
    *   ポスターの色使い（紺色背景、金色の文字）を再現。
2.  **アニメーション演出**:
    *   画面上部から雪が降ってくるような静的なループアニメーション。
3.  **申し込みフォーム**:
    *   入力項目:
        *   お名前 (必須)
        *   参加人数 (必須: 大人/子供の内訳など)
        *   メールアドレス (任意)
    *   送信ボタン: 押下時にバックエンドへデータを送信する。
    *   完了表示: 送信成功時に「登録ありがとうございました」等のメッセージを出す。

#### 【バックエンド（サーバー）】
1.  **受付API (Endpoint)**:
    *   フロントエンドからのデータを受け取る窓口 (URL: `/api/register` 等)。
2.  **データ処理**:
    *   受け取ったJSONデータをJavaのオブジェクトに変換。
    *   **今回は簡易版として**: データベースは使わず、受け取ったデータをサーバーのログ（コンソール）に表示、もしくはCSVファイル等に追記保存する。
3.  **CORS対応**:
    *   別の場所（ブラウザ）からの通信を許可する設定。

# 2. 開発ロードマップ

初心者でも挫折しないよう、機能を切り分けて少しずつ完成させる「アジャイル的」な進め方です。

### 【Phase 0】 開発環境の整備 (所要目安: 1日)
まずは道具を揃えます。ここが一番のエラー発生ポイントなので焦らず行います。
1.  **Java (JDK) のインストール**: コマンド `java -version` で表示されるようにする。
2.  **IDE (IntelliJ IDEA推奨) のインストール**: コードを書くソフトを入れる。
3.  **Gitの初期設定**: GitHubのアカウントを作り、手元のPCと連携させる。
4.  **プロジェクト作成**: Spring Initializr（Webツール）またはIDEから、Spring Bootの空のプロジェクトを作成する。

### 【Phase 1】 静的ページの作成 (フロントエンド基礎)
Javaは一旦忘れ、見た目だけを作ります。
1.  **HTML構造作成**: `index.html` を作成し、文字情報だけを並べる。
2.  **CSSデザイン**: 背景を紺色にし、文字を金色にし、スマホで見やすいように配置を整える（FlexboxやGridを使用）。
3.  **GitHubへPush**: ここで一度GitHubに上げ、コードが保存される流れを確認する。

### 【Phase 2】 動きの実装 (JavaScript)
1.  **降雪アニメーション**: JavaScriptで `div` 要素（雪の粒）を生成し、CSSアニメーションで落下させるロジックを書く。
2.  **フォームの制御**: 送信ボタンが押されたときに、画面がリロードされないようにし（`preventDefault`）、入力された値を取得する処理を書く（コンソールに出すだけ）。

### 【Phase 3】 バックエンドの構築 (Java/Spring Boot)
ここからサーバー側の開発です。
1.  **Controller作成**: URL (`/api/register`) にアクセスが来たら「Hello」と返すだけのプログラムを書く。
2.  **データクラス作成**: `Name`, `Count` などが入る `RegistrationForm` クラス（入れ物）を定義する。
3.  **POST受信処理**: フロントエンドから送られてくるデータを受け取り、IntelliJのログ画面に `System.out.println` で表示させる。

### 【Phase 4】 フロントとバックの結合
1.  **Fetch APIの実装**: JavaScriptから、Phase 3で作ったJavaサーバーへデータを投げる処理を書く。
2.  **CORS設定**: Java側で「このHTMLからの通信は安全だよ」と許可するアノテーションを追加する。
3.  **動作確認**: ブラウザのフォームに入力してボタンを押し、Java側の黒い画面（ログ）に入力内容が表示されれば成功！

### 【Phase 5】 実機テスト (デプロイごっこ)
1.  **ngrokの起動**: コマンド一発でJavaサーバーをインターネットに公開する。
2.  **QRコード生成**: ngrokのURLをQRコードにする。
3.  **スマホでアクセス**: 自分のスマホでWi-Fiを切って（4G/5Gで）アクセスし、申し込みができるか試す。

# 3. 推奨ディレクトリ構成（ファイルの置き場所）

開発が進むとファイルが増えるので、最初からこの形を意識しておくと楽です。
Spring Bootの標準的な構成に準拠しています。

```text
ProjectName/
├── src/
│   ├── main/
│   │   ├── java/com/example/christmas/  <-- Javaのコードはここ
│   │   │   ├── ChristmasApplication.java    (起動用メインファイル)
│   │   │   ├── controller/
│   │   │   │   └── RegistrationController.java (受付窓口)
│   │   │   └── model/
│   │   │       └── UserData.java           (データの型定義)
│   │   │
│   │   └── resources/
│   │       └── static/                  <-- フロントエンドはここに入れると自動公開される
│   │           ├── index.html               (トップページ)
│   │           ├── css/
│   │           │   └── style.css            (デザイン)
│   │           ├── js/
│   │           │   └── script.js            (アニメ・通信)
│   │           └── images/
│   │               └── poster_bg.png        (画像素材など)
│   │
│   └── test/                            (テストコード用・今回は使わない)
├── pom.xml (または build.gradle)        (ライブラリ管理ファイル)
└── .gitignore                           (Gitで無視するファイル設定)
```
